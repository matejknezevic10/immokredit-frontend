Präsentations-Leitfaden — Use Cases

Diese Use Cases zeigen den kompletten Workflow eines Kreditvermittlers — vom ersten Kontakt bis zum Abschluss.

1
Neuer Lead kommt rein
Automatische Erfassung + Zuweisung
1
Kunde füllt Formular auf Website aus (OnePage Funnel)→ Lead wird automatisch erstellt mit Kontaktdaten + Kreditwunsch
2
Lead erscheint im Dashboard mit Ampelstatus (Gelb = neu)→ Temperatur wird automatisch eingestuft (warm/hot/cold)
3
Lead einem Berater zuweisen (Roland, Slaven oder Daniel)→ Berater sieht den Lead in seinem persönlichen Dashboard
Demo-Tipp: Zeig das Dashboard und wie ein neuer Lead automatisch erscheint. Klick auf den Lead und zeig die Detail-Ansicht.
2
Dokumente per Email erhalten
Automatische OCR + Kundenordner
1
Kunde sendet Dokumente per Email (Reisepass, Lohnzettel, etc.)→ Email wird automatisch empfangen und verarbeitet
2
Jeffrey (KI) erkennt Dokumenttyp automatisch→ "Reisepass erkannt", "Lohnzettel erkannt" — Felder werden extrahiert
3
Dokument landet auf Google Drive im Kundenordner→ Nummerierte Ordnerstruktur: "1_Max Mustermann/1. Max Mustermann Reisepass.pdf"
4
Kundendaten werden automatisch befüllt→ Person, Haushalt, Finanzplan — aus OCR extrahierte Felder
Demo-Tipp: Lade ein Dokument über den 📤 Button hoch. Zeig wie Jeffrey es analysiert und die Felder automatisch befüllt werden. Öffne dann Google Drive und zeig den Kundenordner.
3
Unterlagen-Check & Erinnerung
Jeffrey prüft Vollständigkeit
1
Klick auf "📋 Unterlagen prüfen"→ Jeffrey zeigt Checkliste: welche Dokumente vorhanden, welche fehlen
2
Ein-Klick Erinnerungs-Email generieren→ Personalisierte Email an Kunden mit Liste der fehlenden Unterlagen
Demo-Tipp: Öffne einen Kunden mit ein paar Dokumenten. Klick "Unterlagen prüfen" und zeig die Checkliste. Generiere die Erinnerungs-Email.
4
Kennzahlen-Berechnung
DSTI, LTV, Immobilienwert — automatisch
1
Kundendaten werden erfasst (Einkommen, Kredit, Objekt)→ Kennzahlen werden automatisch berechnet
2
DSTI (Schuldendienstquote) — Grün/Gelb/Rot Bewertung→ "Ausgezeichnet", "Akzeptabel", "Kritisch" mit Ampel-Farbe
3
LTV (Beleihungsquote) — mit visueller Skala→ Zeigt Eigenkapitalquote und Finanzierungsrisiko
4
Geschätzter Immobilienwert — basierend auf PLZ, Fläche, Baujahr→ Automatische Schätzung wenn kein Kaufpreis vorhanden
Demo-Tipp: Zeig die Kennzahlen-Karten eines Kunden. Erkläre DSTI und LTV und warum das für die Bankentscheidung wichtig ist.
5
Unterlagen an Bank senden
Verschlüsselter Link oder PDF-Anhang
1
Klick auf "📨 Unterlagen versenden"→ Modal öffnet sich mit 2 Optionen
2a
Option A: Verschlüsselter Link→ Bank erhält 2 separate Emails: Download-Link + Passwort (sicherer)
2b
Option B: PDF-Anhang→ Alle Dokumente direkt als Anhang in einer Email (max. 25MB)
3
Email-Tracking zeigt ob Bank die Email geöffnet hat→ "Gelesen" + Zeitstempel im Email-Verlauf
Demo-Tipp: Sende einen verschlüsselten Link an deine eigene Email. Zeig die 2 Emails (Link + Passwort). Öffne den Link und lade ein Dokument herunter.
6
Digitale Signatur
Kunde unterschreibt online
1
Klick auf "🔗 Signatur-Link erstellen"→ Einmaliger, zeitlich begrenzter Link wird generiert
2
Link per WhatsApp oder Email an Kunden senden→ Kunde öffnet Link auf Handy/Computer
3
Kunde unterschreibt digital (Touch/Maus)→ "✅ Digital signiert" erscheint auf der Kundendetailseite
4
Unterschrift als PDF gespeichert→ In Kundendokumenten + Google Drive (unterschrift_kunde_[timestamp].pdf)
Demo-Tipp: Erstelle einen Signatur-Link, öffne ihn auf deinem Handy und unterschreibe. Geh zurück zur App und zeig den grünen "Digital signiert" Banner.
7
Pipeline & Deal-Management
Vom Lead zum Abschluss
1
Pipeline-Ansicht zeigt alle Deals in Kanban-Stages→ Synchronisiert mit Pipedrive CRM
2
Deal zwischen Stages verschieben→ Unterlagen vollständig → Bank-Anfrage → Warten auf Zusage → Abgeschlossen
3
Kunde archivieren nach Abschluss→ Verschwindet aus aktiver Liste, bleibt im Archiv + Datenbank
Demo-Tipp: Zeig die Pipeline-Ansicht. Erkläre die Stages und wie ein Deal von links nach rechts wandert.
8
Jeffrey KI-Assistent
Chat + OCR + Unterlagen-Check
1
Jeffrey Chat — Fragen zu Kunden, Deals, Dokumenten stellen→ KI kennt alle Daten aus der Datenbank und antwortet auf Deutsch
2
OCR Analyse — Dokumente automatisch auswerten→ Erkennt Dokumenttyp, extrahiert Felder mit Konfidenz-Score
3
Unterlagen-Check — Vollständigkeitsprüfung→ Fehlende Dokumente identifizieren + Erinnerung generieren
Demo-Tipp: Zeig alle 3 Jeffrey-Funktionen nacheinander. Am beeindruckendsten ist die OCR: lade einen Reisepass hoch und zeig wie die Felder automatisch extrahiert werden.

Demo-Skript — Schritt für Schritt

Empfohlener Ablauf für eine 15-20 Minuten Go-Live Präsentation.

Vorbereitung (5 Min vor der Demo)

  • App öffnen und einloggen
  • Ein Test-Dokument bereithalten (z.B. Reisepass-Foto, Lohnzettel-PDF)
  • Google Drive Ordner in separatem Tab öffnen
  • Eigene Email-Inbox in separatem Tab öffnen (für Secure-Link Demo)
  • Handy bereithalten (für Signatur-Demo)

Ablauf der Präsentation

1. Dashboard & Überblick (2 Min)

// Zeige das Dashboard
→ "Das ist unser zentrales Dashboard."
→ "Hier sehe ich meine Kunden mit offenen Aufgaben."
→ "Die Checkboxen zeigen mir sofort welche Daten fehlen."
→ "Oben rechts: Aktivitäten von heute."

2. Neuen Kunden anlegen & Dokument hochladen (4 Min)

// Gehe zu einem bestehenden Kunden oder lege einen neuen an
→ "Hier ist unser Kunde. Ich lade jetzt ein Dokument hoch."
// Klick auf 📤 Hochladen, wähle Datei
→ "Jeffrey erkennt automatisch um was es sich handelt."
// Klick auf 🔍 OCR Analysieren
→ "Die Daten werden direkt in die Kundenfelder übernommen."
// Zeige Person/Haushalt Felder
→ "Und das Dokument ist jetzt auch auf Google Drive."
// Wechsle zum Google Drive Tab

3. Unterlagen prüfen & Erinnerung (2 Min)

// Klick auf 📋 Unterlagen prüfen
→ "Jeffrey prüft welche Unterlagen noch fehlen."
→ "Mit einem Klick generiere ich eine Erinnerungs-Email."
→ "Der Kunde bekommt eine Liste was noch ausständig ist."

4. Kennzahlen zeigen (2 Min)

// Scrolle zu den Kennzahlen
→ "Hier berechnet die App automatisch die Schlüsselzahlen."
→ "DSTI zeigt mir die Schuldendienstquote — hier grün, also alles ok."
→ "LTV zeigt die Beleihungsquote — wichtig für die Bank."
→ "Der geschätzte Immobilienwert wird aus PLZ und Fläche berechnet."

5. Unterlagen an Bank senden (3 Min)

// Klick auf 📨 Unterlagen versenden
→ "Zwei Optionen: Verschlüsselter Link oder PDF-Anhang."
→ "Der verschlüsselte Link ist sicherer — 2 separate Emails."
// Sende an eigene Email-Adresse
→ "Schauen wir in die Inbox..."
// Zeige die 2 Emails (Link + Passwort)
→ "Der Bankberater gibt das Passwort ein und kann die Dokumente downloaden."

6. Digitale Signatur (3 Min)

// Klick auf 🔗 Signatur-Link erstellen
→ "Ich erstelle einen Signatur-Link für den Kunden."
// Kopiere Link und öffne auf Handy
→ "Der Kunde öffnet den Link und unterschreibt."
// Unterschreibe auf Handy, gehe zurück zur App
→ "Die Unterschrift wird als PDF gespeichert — im System und auf Google Drive."

7. Pipeline & Archiv (2 Min)

// Gehe zur Pipeline-Ansicht
→ "In der Pipeline sehe ich alle Deals — synchronisiert mit Pipedrive."
→ "Wenn der Deal abgeschlossen ist, archiviere ich den Kunden."
// Zeige Archiv-Menüpunkt
→ "Im Archiv kann ich jederzeit wieder darauf zugreifen."

8. Abschluss & Fragen (2 Min)

→ "Das war der komplette Workflow: Lead → Dokumente → Analyse → Bank → Signatur → Abschluss."
→ "Alles an einem Ort, alles automatisiert."
→ "Fragen?"
